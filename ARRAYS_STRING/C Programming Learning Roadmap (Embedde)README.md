C Programming Learning Roadmap (Embedded Systems Focus)
📌 Objective

To build strong C programming fundamentals required for Embedded Systems development, with clear understanding of logic, memory behavior, and data representation.

This roadmap focuses on how C works internally, not just syntax or library usage.

🧩 Learning Philosophy

Logic first, syntax second

No shortcut libraries initially

Build concepts step-by-step

Learn in a way that scales to microcontrollers & embedded firmware

✅ LEVEL 1: Arrays (Completed)
Topics Covered

One-dimensional arrays

Array input and output

Copying one array into another

Logical operations on arrays (count, sum, search, etc.)

Key Learnings

Index-based memory access

Importance of array bounds

Off-by-one errors and prevention

🔄 LEVEL 2: Strings (Current Focus)
Topics to Learn

What is a string in C

Character array vs string literal

Null terminator ('\0')

String input/output using %s

Manual string length calculation (without strlen)

Manual string copy logic (without strcpy)

Manual string comparison logic (without strcmp)

Goal

Understand how strings are stored and processed internally, without relying on library shortcuts.

🔢 LEVEL 3: Data Representation
Topics to Learn

ASCII values of characters

Integer vs character representation

Decimal, Hexadecimal, Octal formats

Format specifiers:

%d → decimal

%c → character

%x → hexadecimal

%o → octal

Important Concepts

Difference between '1' and 1

Character ↔ integer conversion

How data is represented in memory

🔍 LEVEL 4: Array vs String (Conceptual Clarity)
Topics to Learn

Difference between integer arrays and character arrays

Why strings require a null terminator

Memory layout differences

Why %s works for strings but %d does not

Proper use cases of arrays vs strings

🧠 LEVEL 5: Arrays & Strings with Functions
Topics to Learn

Passing arrays to functions

Modifying arrays inside functions

Why array size is not automatically known inside functions

String processing using functions

Modular programming using functions

Goal

Build a strong base for pointer concepts.

⚠️ PRE-POINTER CONCEPT (Mandatory Before Pointers)
Call by Value vs Call by Reference (Conceptual)

Why variables do not change inside functions

Why arrays behave differently when passed to functions

Understanding the concept of memory address

No pointer syntax yet — only conceptual clarity

Pointers

Pointer basics

Relationship between pointers and arrays

Pointer arithmetic

Strings using pointers

Structures and pointers

Embedded-style coding patterns
